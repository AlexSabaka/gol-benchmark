# Token Counting - Quick Verification Guide

## ✅ Implementation Status

Token counting is **FULLY IMPLEMENTED and WORKING**!

## How to Verify It Works

### Step 1: Run a New Test

```bash
# Generate a test set
python src/stages/generate_testset.py configs/testsets/your_config.yaml

# Run the test (tokens will be automatically counted)
python src/stages/run_testset.py testsets/testset_*.json.gz \
    --model qwen3:0.6b --provider ollama --output-dir results/test_tokens/

# Generate report with token statistics  
python src/stages/analyze_results.py results/test_tokens/*.json.gz \
    --output reports/token_test.md --visualize
```

### Step 2: Check the Report

Open `reports/token_test.md` and you'll see:

**In the Summary Table:**
```
| Model | ... | Avg Tokens/Test | ...
| qwen3:0.6b | ... | 1.2K | ...
```

**In the Detailed Section:**
```markdown
### Token Usage:
- **Total Input Tokens**: 45,230
- **Total Output Tokens**: 38,150
- **Total Tokens**: 83,380
- **Avg Input per Test**: 452
- **Avg Output per Test**: 382
```

**In the Charts:**
- `token_usage.png`: Bar charts showing input vs output tokens
- `token_efficiency_scatter.png`: Correlation plots (tokens vs accuracy)

## ⚠️ Important Notes

### 1. Old Result Files Don't Have Token Data

**Result files created BEFORE the token counting implementation will show "0" for tokens.**

To get token data, you must:
- ✅ Run NEW tests using the current code
- ❌ Old results will show "0" in the token column

### 2. TUI Doesn't Show Tokens During Execution

The **TUI (benchmark_tui.py) does NOT display token statistics during execution.**

Token information appears **ONLY in the generated reports** (Markdown/HTML files).

To see tokens after running via TUI:
1. Complete the benchmark run
2. Check the generated report files in the output directory
3. Look for the "Avg Tokens" column in tables
4. Look for the "Token Usage" section in detailed reports
5. Check the visualizations folder for token usage charts

### 3. Both Markdown and HTML Reports Show Tokens

**Markdown Reports** (`.md` files):
- Summary table has "Avg Tokens/Test" column
- Detailed sections have "Token Usage" subsection

**HTML Reports** (`.html` files):
- Summary table has "Avg Tokens" column
- Detailed sections have "Token Usage" subsection with metrics

## Troubleshooting

### "I don't see token columns in my reports"

**Cause**: You're looking at reports generated from OLD result files.

**Solution**: Run a new test using the current code.

### "Token columns show 0"

**Cause**: The result files don't have token data (generated before implementation).

**Solution**: Run new tests to generate results with token data.

### "TUI doesn't show tokens"

**Expected behavior**: TUI doesn't display tokens during execution.

**Solution**: Check the generated reports after the run completes.

## Files Modified for Token Counting

1. **src/stages/run_testset.py**
   - Calculates tokens during test execution
   - Stores in result JSON under `tokens` field
   - Adds to `summary_statistics`

2. **src/stages/analyze_results.py**
   - Extracts token data from results
   - Adds token columns to all tables
   - Adds token sections to detailed reports
   - Generates 2 token visualization charts

3. **src/cli/benchmark_tui.py**
   - ❌ NOT MODIFIED (doesn't show tokens during execution)
   - ✓ Reports generated by TUI will have token data
